Group A — Very Short Answer (1 mark each)
(I) Approximation ratio is always ≤ 1 — True or False? 1 mark ▾

FALSE

For minimization problems, the approximation ratio = Algorithm's output / OPT ≥ 1 (since the approximation cannot be better than optimal).
For maximization problems, ratio = OPT / Algorithm's output ≥ 1 (or stated as ≤ 1 if defined inversely). Either way it is NOT always ≤ 1 in the standard definition.

(II) O(2ⁿ) > O(n²) for n > 3 — True or False? 1 mark ▾

TRUE

Exponential growth dominates polynomial growth. For n = 4: 2⁴ = 16 vs 4² = 16 (equal). For n = 5: 32 vs 25. Beyond n = 4, 2ⁿ always exceeds n². The asymptotic class O(2ⁿ) is larger (harder) than O(n²).

(III) Time complexity of Job Sequencing with Deadline (Greedy)? 1 mark ▾

O(n²) — with a simple array-based slot search (for each of n jobs, scan backward through up to n slots).

With Union-Find optimization: O(n log n) dominated by the initial sort.

(IV) Topological sorting is performed on ___ graph. 1 mark ▾

Directed Acyclic Graph (DAG)

It requires: (1) directed edges, and (2) no cycles. The result is a linear ordering of vertices such that for every directed edge u → v, u appears before v.

(V) f(n) = 2n³ + 4n², then f(n) = O(n³) — True or False? 1 mark ▾

TRUE

The dominant term is 2n³. For c = 3, n₀ = 4: 2n³ + 4n² ≤ 3n³ holds for all n ≥ 4. Therefore by Big-O definition, f(n) = O(n³).

(VI) Time complexity of Prim's algorithm (V vertices)? 1 mark ▾

O(V²) using adjacency matrix  |  O(E log V) using binary min-heap (priority queue)

(VII) Best case time complexity of Linear Search? 1 mark ▾

O(1) — the target element is found at the very first position.

(VIII) Time complexity of Kruskal's algorithm (E edges)? 1 mark ▾

O(E log E) — sorting all E edges dominates. Equivalently O(E log V) since E ≤ V². Union-Find operations take nearly O(E·α(V)) ≈ O(E).

(IX) DFS uses ___ data structure. 1 mark ▾

Stack (LIFO — either explicit or via the recursion call stack).

(X) Give two examples of Optimization Problems. 1 mark ▾
  1. Travelling Salesman Problem (TSP) — minimize the total cost of a Hamiltonian tour.
  2. Fractional Knapsack Problem — maximize total profit within a given weight limit.
(XI) Worst case time complexity of Heap Sort? 1 mark ▾

O(n log n) — same for best, average, and worst case. Building the heap: O(n). Extracting n elements: O(n log n).

(XII) Space complexity of Merge Sort? 1 mark ▾

O(n) — auxiliary array of size n is needed during the merge step.

Q2: Little-oh (o) notation + Prove log n! = O(n log n) 5 marks ▾

Little-oh (o) notation:

f(n) = o(g(n)) means: for every positive constant c > 0, there exists n₀ such that 0 ≤ f(n) < c · g(n) for all n ≥ n₀.

Equivalently: lim(n→∞) f(n)/g(n) = 0

It is a strict upper bound — g grows strictly faster than f. Unlike Big-O (which may be tight), little-o is always strict.

Example: n = o(n²) because n/n² = 1/n → 0.

Proof: log n! = O(n log n)

log n! = log 1 + log 2 + log 3 + ... + log n

Each term: log k ≤ log n  for all 1 ≤ k ≤ n

Therefore:
log n! = Σ(k=1 to n) log k ≤ Σ(k=1 to n) log n = n · log n

So: log n! ≤ n · log n

By Big-O definition (c=1, n₀=1):
log n! = O(n log n)  ✓
Q3: Optimization Problem vs Decision Problem + Non-Polynomial algorithm examples 5 marks ▾

Optimization Problem: Find the best feasible solution (maximize or minimize an objective function).
Example: "Find the shortest path from A to B." "Maximize profit in knapsack."

Decision Problem: Answer is YES or NO.
Example: "Is there a path from A to B with cost ≤ k?" Every optimization problem has a corresponding decision version. NP-completeness theory is defined in terms of decision problems.

Two examples of Non-Polynomial (NP) algorithms:

  1. Travelling Salesman Problem (brute force) — checks all n! permutations → O(n! · n), which is super-exponential.
  2. 0-1 Knapsack (brute force) — checks all 2ⁿ subsets of items → O(2ⁿ).
Q4: Properties of an Algorithm 5 marks ▾
  1. Input: An algorithm takes zero or more inputs, externally supplied.
  2. Output: At least one quantity (result) must be produced.
  3. Definiteness: Every step must be precisely and unambiguously defined — no ambiguity about what to do next.
  4. Finiteness: The algorithm must terminate after a finite number of steps for all inputs.
  5. Effectiveness: Each step must be basic, feasible, and executable in a finite time — no abstract steps requiring infinite resources.

Additional properties often expected:

  • Correctness: Produces the right output for all valid inputs.
  • Efficiency: Uses minimal time and space resources.
  • Generality: Works for a range of inputs, not just special cases.

Q6: Recurrence T(n) = 3T(n/4) + cn² using Recursion Tree Method 5 marks ▾

Step 1: Build the recursion tree:

Level 0:        cn²                     (1 node, cost = cn²)
Level 1:   c(n/4)² × 3               = 3cn²/16
Level 2:   c(n/16)² × 9              = 9cn²/256
Level i:   c(n/4ⁱ)² × 3ⁱ            = (3/16)ⁱ × cn²

Step 2: Identify the ratio: ratio r = 3/16 < 1 → convergent geometric series

Step 3: Sum all levels:

Total cost = cn² × Σ(i=0 to ∞) (3/16)ⁱ
           = cn² × 1/(1 − 3/16)
           = cn² × 16/13
           = O(n²)

Step 4: Count levels: Recursion bottoms out when n/4ⁱ = 1 → i = log₄n. Leaves = 3^(log₄n) = n^(log₄3) ≈ n^0.79

Since n^0.79 < n², the root level dominates.

T(n) = O(n²)
Verify with Master Theorem: a=3, b=4, f(n)=cn². n^(log₄3) ≈ n^0.79. Since f(n)=n² = Ω(n^(0.79+ε)), Case 3 applies → T(n)=Θ(n²) ✓

Q7: Quick Sort — Algorithm + Best/Worst/Average complexity 5 + 10 marks ▾

(a) Algorithm:

QUICKSORT(A, low, high):
  if low < high:
    pi = PARTITION(A, low, high)
    QUICKSORT(A, low, pi − 1)
    QUICKSORT(A, pi + 1, high)

PARTITION(A, low, high):
  pivot = A[high]
  i = low − 1
  for j = low to high − 1:
    if A[j] ≤ pivot:
      i = i + 1
      swap(A[i], A[j])
  swap(A[i+1], A[high])
  return i + 1

(b) Time Complexity Analysis:

Best Case — O(n log n): Pivot always splits array in two equal halves.

T(n) = 2T(n/2) + O(n)
By Master Theorem (Case 2): T(n) = O(n log n)

Worst Case — O(n²): Pivot is always the smallest or largest (sorted/reverse-sorted input).

T(n) = T(n−1) + T(0) + O(n) = T(n−1) + O(n)
     = O(n) + O(n−1) + ... + O(1) = O(n²)

Average Case — O(n log n): Pivot is equally likely to be any of the k elements.

T(n) = (1/n) × Σ(k=0 to n−1) [T(k) + T(n−k−1)] + O(n)
Solving: T(n) = O(n log n)

Space: O(log n) average recursion stack, O(n) worst. Not stable. In-place.

Q8: Approximation — Absolute/Relative + Vertex Cover problem + Algorithm + Illustration 4+4+3+4 marks ▾

(a) Absolute vs Relative Approximation:

Absolute Approximation: An algorithm A has absolute approximation bound k if |A(I) − OPT(I)| ≤ k for all instances I. The absolute error is bounded by a constant, regardless of OPT size. Rare — very few problems admit this.

Relative Approximation: An algorithm A has approximation ratio ρ(n) if for all instances I of size n:
For minimization: A(I)/OPT(I) ≤ ρ(n)
For maximization: OPT(I)/A(I) ≤ ρ(n)
where ρ(n) ≥ 1. Most practical approximation algorithms are of this type.

(b) Vertex Cover Problem:

A vertex cover of graph G=(V,E) is a subset C ⊆ V such that for every edge (u,v) ∈ E, at least one of u or v is in C.

The Minimum Vertex Cover Problem: find the smallest such C. It is NP-hard.

Example: In a triangle A−B−C, the set {A,B} is a vertex cover (covers edge AB, edge BC via B, edge AC via A). The minimum vertex cover has size 2.

(c) 2-Approximation Algorithm for Vertex Cover:

APPROX-VERTEX-COVER(G):
  C = {}                    // empty cover
  E' = copy of G.E          // working set of edges
  while E' is not empty:
    let (u, v) be any edge in E'
    C = C ∪ {u, v}          // add both endpoints
    remove from E' all edges
      incident on u or v
  return C

Why it's a 2-approximation: The edges selected form a matching (no two share an endpoint). Every matching edge requires at least one endpoint in OPT. So |OPT| ≥ |matching|, while |C| = 2×|matching|. Hence |C| ≤ 2×|OPT|.

(d) Illustration: For graph with vertices A,B,C,D,E,F,G:
The algorithm picks an uncovered edge, adds both its endpoints to C, removes all incident edges, and repeats until all edges are covered. The resulting C satisfies |C| ≤ 2 × |optimal vertex cover|.

Q9: Big-O proofs — sum rule, product rule + x^n in O(log n) 4+4+4+3 marks ▾

(a) Prove f₁(n) + f₂(n) = O(max(g₁(n), g₂(n))):

Given: f₁(n) = O(g₁(n)) → ∃ c₁, n₁: f₁(n) ≤ c₁·g₁(n) ∀n ≥ n₁
       f₂(n) = O(g₂(n)) → ∃ c₂, n₂: f₂(n) ≤ c₂·g₂(n) ∀n ≥ n₂

For n ≥ max(n₁, n₂):
f₁(n) + f₂(n) ≤ c₁·g₁(n) + c₂·g₂(n)
             ≤ c₁·max(g₁,g₂) + c₂·max(g₁,g₂)
             = (c₁+c₂) · max(g₁(n), g₂(n))

Let c = c₁+c₂, n₀ = max(n₁,n₂).
∴ f₁(n)+f₂(n) = O(max(g₁(n), g₂(n)))  ✓

(b) Prove f₁(n) × f₂(n) = O(g₁(n) × g₂(n)):

f₁(n) ≤ c₁·g₁(n)  ∀n ≥ n₁  (from f₁ = O(g₁))
f₂(n) ≤ c₂·g₂(n)  ∀n ≥ n₂  (from f₂ = O(g₂))

Since all functions are non-negative:
f₁(n) × f₂(n) ≤ c₁·g₁(n) × c₂·g₂(n)
              = (c₁·c₂) × g₁(n)×g₂(n)

Let c = c₁×c₂, n₀ = max(n₁,n₂).
∴ f₁(n)×f₂(n) = O(g₁(n)×g₂(n))  ✓

(c) Algorithm to compute xⁿ in O(log n) [n is an integral power of 2]:

POWER(x, n):
  if n == 1:
    return x
  half = POWER(x, n/2)    // since n is power of 2, n/2 is exact
  return half × half

(d) Time complexity is O(log n):

Recurrence: T(n) = T(n/2) + O(1)

Using Master Theorem: a=1, b=2, f(n)=O(1)
n^(log₂1) = n⁰ = 1 = Θ(1) = Θ(f(n))  → Case 2
T(n) = Θ(log n) = O(log n)  ✓

Intuitively: we halve n each time → log₂n recursive calls.
Q10: Hamiltonian Cycles (backtracking) + TSP Branch & Bound + Bin-Packing 4+6+5 marks ▾

(a) Hamiltonian Cycle Algorithm using Backtracking:

HAMILTONIAN(G, path[], pos):
  if pos == V:                        // all vertices included
    if edge(path[pos−1], path[0]):    // can close the cycle?
      print path[] + path[0]          // print the Hamiltonian cycle
    return
  for v = 1 to V:
    if isSafe(v, path, pos, G):
      path[pos] = v
      HAMILTONIAN(G, path, pos+1)
      path[pos] = −1                  // BACKTRACK

isSafe(v, path, pos, G):
  if no edge from path[pos−1] to v: return false
  for i = 0 to pos−1:
    if path[i] == v: return false     // already visited
  return true

Main:
  path[0] = 0  (start from vertex 0)
  HAMILTONIAN(G, path, 1)

(b) TSP using Branch and Bound:

For the given graph (A,B,C,D with edge weights), the steps are:

1.Write the cost matrix. Set diagonal and already-forbidden edges to ∞.
2.Row reduction: Subtract the minimum of each row from all entries in that row. Sum all minima = row reduction cost.
3.Column reduction: Subtract the minimum of each column. Sum all minima = column reduction cost.
4.Lower bound at root = row reduction cost + column reduction cost.
5.Branch on each edge: for edge (i,j), set row i and col j to ∞, set (j,i)=∞ (no reverse), re-reduce matrix. New bound = parent bound + cost(i,j) + new reductions.
6.Use Best-First (lowest lower bound) or DFS. Prune nodes whose bound ≥ current best complete tour.
7.When a leaf (complete tour) is found, update the best solution. Return the minimum cost tour.
For the graph in the paper: Start → compute cost matrix from the given edges (weights 2,3,4,5,6,7,10,11,12,18). Apply row+column reduction. Expand the most promising branch at each step.

(c) Bin-Packing Problem:

Given n items with sizes s₁, s₂, …, sₙ (each 0 < sᵢ ≤ 1) and bins of unit capacity 1, pack all items into the minimum number of bins.

It is NP-hard. Common approximation strategies:

AlgorithmStrategyRatio
Next Fit (NF)Place in current bin if fits, else open new bin≤ 2 × OPT
First Fit (FF)Place in first bin with enough space≤ 1.7 × OPT
First Fit Decreasing (FFD)Sort items descending, then apply FF≤ (11/9) × OPT + 6/9
Q11: Graph Coloring — definition + algorithm + polynomial? + trace with 3 colors 2+5+3+5 marks ▾

(a) Graph Coloring Problem:

Assign colors from a set {1, 2, …, m} to vertices of G=(V,E) such that no two adjacent vertices share the same color. The chromatic number χ(G) is the minimum m for which this is possible. The decision version: "Can G be colored with ≤ k colors?" is NP-complete for k ≥ 3.

(b) Backtracking Algorithm for Graph Coloring:

GRAPHCOLOR(G, k, color[], v):
  if v == V:                      // all vertices colored
    print color[]
    return
  for c = 1 to k:
    if isSafe(G, color, v, c):
      color[v] = c
      GRAPHCOLOR(G, k, color, v+1)
      color[v] = 0                // BACKTRACK (uncolor)

isSafe(G, color, v, c):
  for each neighbor u of v in G:
    if color[u] == c: return false  // conflict
  return true

Main:
  color[] = {0, 0, ..., 0}       // all uncolored
  GRAPHCOLOR(G, k, color, 0)

(c) Is it solvable in polynomial time?

No. Graph coloring (for k ≥ 3) is NP-complete. The backtracking algorithm has exponential worst-case time complexity O(k^V · V) since we try up to k colors for each of V vertices. No polynomial-time algorithm is known. Even determining if a graph is 3-colorable is NP-complete (proven via reduction from SAT).

(d) All solutions for the triangle graph (vertices 1-2-3, fully connected) with 3 colors:

Since all three vertices are mutually adjacent (K₃), all three must get different colors. With colors {R, G, B}:

Vertex 1Vertex 2Vertex 3

GBR
BRG
BGR
Total valid colorings = 3! = 6. Chromatic number χ(K₃) = 3.

Q4: Floyd-Warshall Algorithm for all-pairs shortest path5 marks▾

Algorithm:

FLOYD-WARSHALL(W, n):
  // W is n×n weight matrix; W[i][i]=0, W[i][j]=∞ if no edge
  D = W                         // D will hold shortest distances
  P = adjacency matrix (for path reconstruction, optional)
  for k = 1 to n:               // intermediate vertex
    for i = 1 to n:             // source
      for j = 1 to n:           // destination
        if D[i][k] + D[k][j] < D[i][j]:
          D[i][j] = D[i][k] + D[k][j]
  return D

Key idea: D[i][j] = shortest path from i to j using only vertices {1,...,k} as intermediates.

Time: O(V³)  |  Space: O(V²)

Handles negative edge weights. Detects negative cycles if D[i][i] < 0 after completion.

Used for: all-pairs shortest path, transitive closure (replace + with OR, min with AND).

Q6: Randomized Algorithms — short note5 marks▾

A randomized algorithm uses random number generation to make decisions during execution. The output or runtime may vary between runs on the same input.

Two types:

TypeOutputRuntimeExample
Las VegasAlways correctRandom (expected poly)Randomized QuickSort, Randomized Search
Monte CarloPossibly incorrect (small prob)Deterministic (fixed)Miller-Rabin primality test, Monte Carlo π estimation

Advantages: Simple implementation; avoids worst-case adversarial inputs; often matches or beats deterministic algorithms on average; useful where deterministic polynomial solutions are unknown.

Example — Randomized QuickSort: Choose pivot randomly → expected O(n log n) always, even on sorted input.

(a) Matrix Chain Multiplication — DP:

Dimension array: p = [10, 20, 50, 1, 100]. m[i][j] = min multiplications to compute Aᵢ…Aⱼ

Base: m[i][i] = 0 for all i.

l=2 (chains of 2):
m[1][2] = p[0]·p[1]·p[2] = 10×20×50 = 10000

(b) Bellman-Ford Algorithm (negative edges allowed):

BELLMAN-FORD(G, s):
  dist[s] = 0; dist[v] = ∞ for all v ≠ s
  pred[v] = NULL for all v

  for i = 1 to |V| − 1:                 // V−1 iterations
    for each edge (u, v, w) in E:
      if dist[u] + w < dist[v]:
        dist[v] = dist[u] + w           // relax edge
        pred[v] = u

  // Check for negative cycle
  for each edge (u, v, w) in E:
    if dist[u] + w < dist[v]:
      print "Negative cycle detected"
      return false

  return dist[], pred[]

Time: O(VE). Space: O(V). Correct answer guaranteed after V−1 passes if no negative cycle.

Q10: P / NP / NP-hard / NP-complete + Reducibility + P=NP implications7+3+5 marks▾

(a) P, NP, NP-hard, NP-complete:

ClassDefinitionExamples
PDecision problems solvable by a deterministic TM in polynomial time.Sorting, MST, shortest path, max flow
NPDecision problems whose solutions can be verified in polynomial time. P ⊆ NP.SAT, Hamiltonian cycle, 3-coloring
NP-hardAt least as hard as the hardest NP problems. Every NP problem reduces to it in poly time. May not be in NP.TSP optimization, Halting problem

Diagram: P ⊆ NP. NP-complete ⊆ NP. NP-hard ⊇ NP-complete. If P≠NP, P and NP-complete are disjoint.

(b) Reducibility:

Problem A is polynomial-time reducible to problem B (written A ≤ₚ B) if there exists a polynomial-time transformation f such that any instance x of A maps to instance f(x) of B where: x ∈ A ⟺ f(x) ∈ B.

Implications: If B ∈ P and A ≤ₚ B → A ∈ P. If A is NP-hard and A ≤ₚ B → B is NP-hard. Used to prove NP-completeness.

(c) If P = NP:

  • Every problem whose solution is verifiable in poly time would also be solvable in poly time.
  • All NP-complete problems (SAT, TSP, etc.) would have polynomial solutions.
  • Cryptography collapses: RSA, AES security relies on factoring/discrete log being hard — these would become easy.
  • AI and optimization: protein folding, drug discovery, scheduling would be solved optimally and quickly.
  • Currently unproven; Clay Millennium Prize ($1M) awaits a proof or disproof. Most believe P ≠ NP.
Q11: Flow Network + Augmenting Path/Residual Capacity + Ford-Fulkerson Algorithm5+5+5 marks▾

(a) Flow Network and constraints:

A flow network G=(V,E) is a directed graph with: a source s (no incoming edges from network) and a sink t (no outgoing edges to network). Each edge (u,v) has capacity c(u,v) ≥ 0.

A flow f must satisfy three constraints:

  1. Capacity constraint: 0 ≤ f(u,v) ≤ c(u,v) for all (u,v) ∈ E
  2. Flow conservation: For all v ≠ s,t: Σf(u,v) = Σf(v,w) (inflow = outflow)
  3. Skew symmetry: f(u,v) = −f(v,u)

Value of flow: |f| = Σ f(s,v) for all v.

(b) Augmenting Path and Residual Capacity:

Residual capacity: cₓ(u,v) = c(u,v) − f(u,v). The residual graph Gₓ contains edge (u,v) if cₓ(u,v) > 0, plus reverse edge (v,u) with cₓ(v,u) = f(u,v).

Augmenting path: A simple path from s to t in Gₓ where every edge has positive residual capacity. The bottleneck = min cₓ on the path. Sending bottleneck flow along this path increases total flow.

Max-flow Min-cut theorem: Maximum flow value = minimum cut capacity.

(c) Ford-Fulkerson Algorithm:

FORD-FULKERSON(G, s, t):
  Initialize f(u,v) = 0 for all edges (u,v)

  while ∃ augmenting path p from s to t in residual graph Gf:
    cₓ(p) = min{cₓ(u,v) : (u,v) in p}   // bottleneck
    for each edge (u,v) in p:
      f(u,v) = f(u,v) + cₓ(p)            // send flow
      f(v,u) = f(v,u) − cₓ(p)            // update reverse

  return f    // maximum flow

Time: O(E × |f*|) where f* = max flow. If BFS used (Edmonds-Karp): O(VE²).

TRUE — Brute force exhaustively searches all possible solutions, so it always finds the optimal. It guarantees the best quality — at the cost of exponential or worse time complexity.

Q4: Prim's Algorithm (Greedy) for MST — algorithm + time complexity5 marks▾
PRIMS-MST(G, start):
  key[v] = ∞ for all v; key[start] = 0
  parent[v] = NULL for all v
  inMST[v] = false for all v
  MST = {}

  for i = 1 to V:
    u = vertex with minimum key[] not yet in MST
    inMST[u] = true
    MST.add(u, parent[u])

  return MST

Q5: Cook's Theorem + Clique definition with example5 marks▾

(a) Cook's Theorem (1971):

The Boolean Satisfiability Problem (SAT) is NP-complete. Specifically: Every problem in NP can be polynomial-time reduced to SAT.

SAT: Given a boolean formula in CNF (Conjunctive Normal Form), is there an assignment of variables that makes it TRUE?

Significance: SAT was the first problem proven NP-complete. All subsequent NP-completeness proofs reduce from SAT (or a derived problem like 3-SAT).

(b) Clique:

A clique in undirected graph G=(V,E) is a subset C ⊆ V such that every two distinct vertices in C are connected by an edge — i.e., C induces a complete subgraph.

k-Clique problem: Does G contain a clique of size k? This is NP-complete.

Example: In graph with vertices {A,B,C,D} and edges {AB, AC, BC, BD}:

  • {A,B,C} is a clique of size 3 (all pairs connected: AB, AC, BC ✓)
  • {A,B,C,D} is NOT a clique (AD and CD edges missing)
  • Maximum clique = {A,B,C}, size 3
Q6: Brute Force Algorithm + Approximation Algorithm5 marks▾

(i) Brute Force Algorithm:

An exhaustive search strategy that tries every possible solution and selects the best. It does not exploit any structure of the problem.

Characteristics: Always produces the optimal solution. Very slow — typically O(n²), O(n!), or O(2ⁿ). Useful as a baseline or when n is small.

Examples: Finding all permutations for TSP (O(n!)), checking all pairs for closest pair (O(n²)), sequential search.

(ii) Approximation Algorithm:

A polynomial-time algorithm that finds a solution provably close to optimal for NP-hard problems. Trade-off: gives up optimality to gain tractability.

Characterized by approximation ratio ρ(n): the worst-case ratio of algorithm output to OPT.

Examples: 2-approximation for Vertex Cover, FFD for Bin-Packing (11/9 ratio), greedy (1−1/e) for Set Cover.

(b) N-Queens Algorithm using Backtracking:

NQUEENS(board[], row, n):
  if row == n:
    PRINT(board)    // found a valid solution
    return
  for col = 0 to n−1:
    if isSafe(board, row, col, n):
      board[row] = col          // place queen
      NQUEENS(board, row+1, n)
      board[row] = −1           // BACKTRACK (remove queen)

isSafe(board, row, col, n):
  for i = 0 to row−1:
    if board[i] == col:                   return false // same column
    if |board[i]−col| == |i−row|:        return false // same diagonal
  return true

(c) Time complexity of N-Queens: O(n!) in the worst case (no pruning). In practice much better due to early backtracking.

(d) Floyd-Warshall (All-Pairs Shortest Path):

FLOYD-WARSHALL(D, n):
  // Initialize D from adjacency matrix
  for k = 1 to n:
    for i = 1 to n:
      for j = 1 to n:
        D[i][j] = min(D[i][j], D[i][k] + D[k][j])
  return D

Time: O(V³). Handles negative edges (not negative cycles). D[i][i] < 0 after completion indicates negative cycle.

KRUSKALS(G):

(a) Transitive closure from vertex A:

Apply DFS or BFS from A. Mark all reachable vertices. From the graph: A→B, A→C, A→D (direct). Then trace from B: B→E, B→F. From E: E→G. From F: F→G. From D: D→H (if edge exists). From G→H etc.

(b) Graph Terminology Definitions:

(i) Directed graph (Digraph): G=(V,E) where each edge (u,v) has a direction from u to v. Edge (u,v) ≠ edge (v,u). Represented by arrows.

(ii) Undirected graph: G=(V,E) where edges have no direction. {u,v} = {v,u}. Represented by lines.

(iii) In-Degree and Out-Degree: In-degree of v = number of edges directed INTO v. Out-degree = number directed OUT from v. For undirected graphs, just "degree" = number of incident edges.

(iv) Complete Graph (Kₙ): Every pair of distinct vertices is connected by exactly one edge. Kₙ has n(n−1)/2 edges. Example: K₃ = triangle.

(v) Articulation Point (Cut Vertex): A vertex v in a connected graph G whose removal (along with its edges) increases the number of connected components. Critical for network connectivity analysis.

(vi) Bridge (Cut Edge): An edge (u,v) whose removal disconnects the graph (increases connected components). A bridge is never part of any cycle. Example: In a tree, every edge is a bridge.

(b) Master's Theorem — T(n) = aT(n/b) + f(n):

Three cases based on comparison of f(n) with n^(log_b a):

  • Case 3: f(n) = Ω(n^(log_b(a)+ε)) and regularity holds → T(n) = Θ(f(n))

(c) Little omega (ω) notation:

f(n) = ω(g(n)) means: for every positive constant c > 0, there exists n₀ such that f(n) > c · g(n) for all n ≥ n₀.

Equivalently: lim(n→∞) f(n)/g(n) = ∞  (f grows strictly faster than g)

Relation: f(n) = ω(g(n)) ⟺ g(n) = o(f(n))

Example: n² = ω(n log n) because n²/(n log n) = n/log n → ∞ as n → ∞. ✓
